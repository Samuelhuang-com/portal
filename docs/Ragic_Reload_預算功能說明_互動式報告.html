Ragic 請購單「Reload 預算」功能說明

用於報告、教育訓練與跨部門討論：說明請購單如何依「部門 + 會科 + 預算取用月份」自動從預算管理表單抓取預算金額，並回填至請購單「預算」欄位。

🔄 Reload 預算 📊 預算管理連動 ✅ 部門 + 會科比對 🗓️ 支援指定月份
Purpose

一、功能目的

🎯 自動帶入預算

使用者填寫請購單後，系統依條件到「預算管理」表單抓取對應月份預算。

🔍 降低人工查表

避免使用者手動查 Excel 或手動輸入預算，降低錯誤與溝通成本。

🧭 支援前月預算

若 4 月或更早月份預算尚未使用，可由「預算取用月份」指定來源月份。

核心比對條件：請購單.部門 + 請購單.會科 + 預算取用月份
Logic

二、功能執行邏輯

讀取請購單

取得編號、部門、會科與預算取用月份。

查詢預算管理

依部門與會計科目比對預算管理資料。

取得月份金額

依選擇月份抓 1 月至 12 月對應欄位。

回填預算欄位

將金額寫回請購單「預算」欄位並顯示結果。

請購單：部門 + 會科 + 預算取用月份
        ↓
預算管理：部門 + 會計科目 + 對應月份欄位
        ↓
回填：請購單「預算」欄位
Required Conditions

三、請購單填寫必要條件

1. 部門必須選擇正確

請購單上的「部門」必須與「預算管理」中的部門名稱一致。

正確範例 可能失敗範例
管理 管理部、管理課、管理部門
2. 會科必須與預算管理完全一致

請購單「會科 / 科目」必須與預算管理「會計科目」完全一致。

正確 錯誤或可能比對不到
清潔費用品 清潔用品費、清潔費、清潔用品
3. 預算取用月份要先確認

若目前是 5 月，但 4 月預算尚未使用完，可選擇 4月 後再按 Reload 預算。

建議選項：本月、上月、1月、2月、3月、4月、5月、6月、7月、8月、9月、10月、11月、12月。

4. 新增請購單時需先儲存

Ragic JS Workflow 按鈕通常需要針對已存在的資料執行。新增資料尚未儲存時，可能沒有正式 record id，按鈕可能不會出現或無法正常執行。 

新增請購單 → 填寫部門 / 會科 / 月份 → 儲存 → 按 Reload 預算
Budget Master Data

四、預算管理資料維護注意事項

部門與會計科目要統一

建議部門與會計科目都使用固定選單或主檔，不建議自由輸入，以免出現多種名稱造成比對失敗。

每月預算欄位應完整維護

1 月至 12 月預算欄位都應維護。若無預算，請填 0，不要留空白。

預算資料異動後需重新 Reload

若預算管理資料有調整，既有請購單上的預算欄位不會自動同步，使用者需再次按下 Reload 預算。

Troubleshooting

五、常見錯誤與原因

系統訊息 可能原因 處理方式
找不到符合的預算管理資料 部門或會科名稱不一致 檢查請購單與預算管理的部門、會科文字
該月份預算金額為空 預算管理該月份沒有填金額 補上金額，若無預算請填 0

Operation Flow

六、建議使用流程

一般情境：本月預算

確認部門、會科正確，預算取用月份選「本月」，儲存後按 Reload 預算。

特殊情境：前月預算

若 4 月預算尚未使用，可選擇 4 月後按 Reload 預算，系統會抓 4 月金額。

資料異動：重新載入

預算管理資料修正後，請購單需重新按 Reload 預算才會更新。

Executive Summary

八、簡短版說明

Reload 預算功能會依請購單的「部門 + 會科 + 預算取用月份」到「預算管理」表單查詢對應月份的預算金額，並自動回填至請購單的「預算」欄位。 
使用前請確認：
1. 請購單部門選擇正確
2. 請購單會科與預算管理的會計科目完全一致
3. 預算管理已建立對應部門與會科
4. 對應月份已有預算金額，若無預算請填 0
5. 新增請購單需先儲存後再執行 Reload 預算
6. 若要使用前月或指定月份預算，請先選擇「預算取用月份」
