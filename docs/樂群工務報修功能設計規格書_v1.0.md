# 樂群工務報修功能設計規格書

**版本：v1.0**
**日期：2026-04-15**
**專案：維春集團管理 Portal**
**模組路徑：`/luqun-repair/dashboard`**

---

## 1. 功能概述

「樂群工務報修」是 Portal 的一個完整管理模組，整合春大直商場的報修管理業務。
所有資料來源均為 Ragic「春大直 - 報修清單總表」，透過共用 Service Layer 抓取、清洗、統計後呈現。

### 目的
- 讓主管可一鍵查看本月報修 KPI、趨勢圖表與待處理事項
- 提供 4 張標準統計表（4.1~4.4）供定期檢討使用
- 提供可搜尋、可匯出的明細資料表

## 3. 欄位對應規格

以下為 Ragic 欄位中文名稱對應到 Portal 標準化欄位的映射。如 Ragic 欄位名有變動，請修改 `luqun_repair_service.py` 中的 `RK_*` 常數。

| Portal 標準欄位 | Ragic 欄位名（預設） | 備用欄位名 |
|----------------|---------------------|-----------|
| `case_no` | 報修編號 | 編號 / 案件編號 |
| `title` | 標題 | 報修名稱 / 名稱 |
| `reporter_name` | 報修人姓名 | 報修人 / 申報人 |
| `repair_type` | 報修類型 | 類型 / 類別 |
| `floor` | 發生樓層 | 樓層 / 位置 |
| `occurred_at` | 發生時間 | 報修時間 / 申報時間 |
| `responsible_unit` | 負責單位 | 負責人 / 承辦單位 |
| `work_hours` | 花費工時 | 工時 |
| `status` | 處理狀況 | 狀態 / 進度 |
| `outsource_fee` | 委外費用 | 外包費用 |
| `maintenance_fee` | 維修費用 | 費用 |
| `acceptor` | 驗收者 | 驗收人 |
| `accept_status` | 驗收 | 驗收結果 / 驗收狀態 |
| `closer` | 結案人 | 結案者 |
| `deduction_item` | 扣款事項 | — |
| `deduction_fee` | 扣款費用 | — |
| `finance_note` | 財務備註 | 備註 |
| `completed_at` | 結案時間 | 結案日期 → 驗收日期（依序備用） |

### 衍生欄位說明

| 衍生欄位 | 計算方式 |
|---------|---------|
| `total_fee` | `outsource_fee + maintenance_fee` |
| `is_completed` | `status in COMPLETED_STATUSES`（見下方） |
| `close_days` | `(completed_at - occurred_at).days`（四捨五入至小數第 2 位） |
| `year` / `month` | 從 `occurred_at` 解析 |
| `is_room_case` | 發生樓層/標題含客房關鍵字或可解析出房號 |
| `room_no` | 從 `floor`/`title` 解析 3~4 位數字 |
| `floor_normalized` | 標準化樓層格式（如 `5F`） |

## 4. 完成狀態判定

以下狀態值視為「已完成」（集中配置於 `COMPLETED_STATUSES` 常數，改此處即全模組生效）：

```
已驗收 / 已結案 / 結案 / 完修 / 已完成 / 完成
```

其他值（如「待維修」「處理中」「待驗收」「待協調」「待排除」等）視為未完成。

---

## 5. 案件大小型分類邏輯（4.2 用）

初版邏輯（集中於 `is_large_repair()` 函式，後續可改為依「核可層級」判斷）：

- **小型報修**：`total_fee == 0`（無費用）
- **中大型報修**：`total_fee > 0`（有費用）

## 6. 報修類型標準化 mapping

Ragic 的「報修類型」欄位可能有不同寫法，系統透過 `REPAIR_TYPE_MAPPING` 關鍵字 mapping 到 15 個標準類別：

| # | 標準類別 | 代表關鍵字 |
|---|---------|-----------|
| 1 | 建築 | 建築、結構、外牆、玻璃、電梯 |
| 2 | 衛廁 | 衛廁、馬桶、洗手、烘手機 |
| 3 | 消防 | 消防、偵煙、灑水、鐵捲門 |
| 4 | 空調 | 空調、冷氣、送風機、冷卻 |
| 5 | 機電 | 機電、機房、發電機 |
| 6 | 給排水 | 漏水、水塔、污水 |
| 7 | 排煙 | 排煙、靜電機、截油槽 |
| 8 | 監控 | CCTV、監控、監視 |
| 9 | 弱電 | 交換機、電話、網路 |
| 10 | 照明 | 燈、燈具 |
| 11 | 停車 | 車牌、柵欄機、繳費機 |
| 12 | 其他 | 人流計數器、入金機 |
| 13 | 專櫃 | 專櫃、租戶 |
| 14 | 公區 | 公區、大廳、梯廳、廣場 |
| 15 | 後勤空間 | 辦公室、儲藏、員工餐 |

無法 mapping 的資料保留原始值；完全空白則歸為「其他」。

## 8. Dashboard 規格

### KPI 統計卡（6 項）

| 指標 | 說明 |
|------|------|
| 本月報修件數 | `occurred_at` 落在查詢月的案件數 |
| 已完成件數 | 上述中，`is_completed = true` 的數量 |
| 未完成件數 | 上述中，`is_completed = false` 的數量 |
| 平均結案天數 | 完成案件的 `close_days` 平均值 |
| 費用總額 | 查詢月所有案件的 `total_fee` 合計 |
| 客房報修件數 | 查詢月中，`is_room_case = true` 的數量 |

### 圖表

| 圖表 | 類型 | 說明 |
|------|------|------|
| 近 12 個月趨勢 | 折線圖（recharts LineChart） | 每月報修件數 vs 完成件數 |
| 報修類型分布 | 圓餅圖（PieChart） | 15 類型比例 |
| 發生樓層分布 | 橫條圖（BarChart） | Top 10 樓層 |
| 處理狀況分布 | 橫條圖 | 各狀態件數（顏色對應狀態） |

### 快速摘要區（Top 10 清單）

- 未完成案件 Top 10（依發生時間升排）
- 高費用案件 Top 10（依 total_fee 降排）
- 高工時案件 Top 10（依 work_hours 降排）

## 9. 4.1 報修統計規格

表格格式：統計項目（左）× 1~12 月（欄）

| 行號 | 統計項目 | 公式說明 |
|------|---------|---------|
| ① | 上月累計未完成項目數 | 截至上月底，`is_completed = false` 的累計案件數 |
| ② | 上月未完成，本月結案數 | ①中，`completed_at` 落在本月的案件數 |
| ③ | 累計項目完成率（%） | 截至本月底：累計完成數 / 累計總數 × 100 |
| ④ | 本月報修項目數 | `occurred_at` 落在本月的案件數 |
| ⑤ | 本月報修項目完成數 | ④中，`is_completed = true` 的數量 |
| ⑥ | 本月報修項目完成率（%） | ⑤ / ④ × 100 |

- 查詢指定月份時，該月欄位高亮底色（`#E6F7FF`）
- 完成率欄位顏色：≥80% 綠色、50~79% 橙色、<50% 紅色

|------|------|

## 11. 4.3 報修類型規格

表格格式：類別（左）× 1~12 月（欄）

額外欄位：

- 合計（年度累計）

## 12. 4.4 客房報修表規格

### 查詢條件
- 年 + 月（必選）
- 樓層篩選（可選，預設全部）

### 表格矩陣
- 縱軸：房號（解析自發生樓層/標題）
- 橫軸：固定 12 個客房類別

```
客房房門 / 客房消防 / 客房設備 / 客房傢俱 / 客房燈 / 客房牆
面盆/台面 / 浴廁 / 浴間 / 天地壁 / 配電盤 / 環境
```

### 房號解析規則
1. 從 `floor`/`title` 中搜尋 3~4 位數字（`[5-9]\d{2}` 或 `10\d{2}`）
2. 判定客房條件：發生樓層含 5F~10F 或「客房」關鍵字
3. 無法判定的案件顯示在「未判定房號」區塊，不丟失

### 資料格呈現
- 格子顯示 title 前 8 字（超出顯示「…」）
- 多筆時顯示前 2 筆 + 數量 badge（`+N`）
- Tooltip 顯示完整 title 和狀態

## 13. 明細資料表規格

### 查詢條件
- 年、月、報修類型、發生樓層、處理狀況（下拉選單，選項動態從資料取得）
- 關鍵字（搜尋：報修編號 / 標題 / 報修人姓名，Enter 鍵或查詢按鈕觸發）

### 表格功能
- 分頁（每頁 50 筆）
- 排序（發生時間、費用、工時、結案天數）
- 點擊列開啟詳情 Drawer（顯示全部欄位）
- 狀態欄顏色標示
- 費用欄千分位格式（`$N,NNN`）
- 匯出 Excel（`GET /api/v1/luqun-repair/export?...`，串流回傳 .xlsx）

## 17. 欄位確認方式

首次啟動後，請前往：

```
GET /api/docs → /api/v1/luqun-repair/raw-fields
```

回傳第一筆 Ragic 資料的所有欄位 key 名稱，與 `luqun_repair_service.py` 中 `RK_*` 常數對照確認。
如有不符，修改對應的 `RK_ALIASES` 陣列即可，不需修改業務邏輯程式碼。
