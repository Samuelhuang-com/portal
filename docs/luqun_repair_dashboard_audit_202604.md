## 二、各 KPI 計算公式

### 2.1 本月相關案件（56 件）

```
Step 1 — 上期未結 = 所有 occ_month <= 2026/3 且 status NOT IN COMPLETED_STATUSES 的案件（共 5 件）
Step 2 — 本月新報修 = occ_year=2026, occ_month=4 的案件（共 51 件）
Step 3 — this_month_cases = 上期未結（5）+ 本月新報修（51）= 56 件
```

**完成狀態集合（COMPLETED_STATUSES）：**  
`已驗收、已結案、結案、完修、已完成、完成、已辦驗`

**排除狀態（EXCLUDED_STATUSES）：**  
`取消`（完全不計入任何統計，明細總表仍保留）

### 2.2 三類互斥分類

```
已完成（34）：status IN COMPLETED_STATUSES
待辦驗（5）： status == "待辦驗"（不在 COMPLETED_STATUSES 內）
未完成（17）：total - 已完成 - 待辦驗 = 56 - 34 - 5 = 17
三類合計：34 + 5 + 17 = 56 ✅
```

### 2.3 平均結案天數（8.4 天）

```
close_days（每筆）= (completed_at - occurred_at) / 86400（天，小數）
條件：is_completed=True AND occurred_at IS NOT NULL AND completed_at IS NOT NULL
avg = SUM(close_days) / 34 = 8.364

顯示：後端 round(2) → 8.36；前端顯示 round(1) → 8.4 天
日期欄位：occurred_at（報修日期，含時分）、completed_at（完工時間優先，fallback 結案時間、驗收時間）
```

### 2.4 本月工時統計（42.50 hr）

```
total_work_hours = SUM(work_hours) for this_month_cases（56件）
work_hours 來源（Ragic alias 優先順序）：
  1. 花費工時（直接 hr，人工填寫）
  2. 工務處理天數 × 24（fallback，Excel 無此欄）
  3. 預估工時（次備用）

注意：Excel「待修工時」= 報修時間到現在的 elapsed hours，非實際工時，不可比對。
Excel「維修工時」欄位的 56 件合計僅 0.02 hr（幾乎全空），無法比對。
```

### 2.5 費用口徑說明

```
統計月份（stat_year / stat_month）決定方式：
  已完成案件 → stat_month = completed_at 的年月（結案月份）
  未完成案件 → stat_month = occurred_at 的年月（報修月份）

扣款費用計入條件（雙重）：
  ① is_completed(status) = True
  ② deduction_counter_name 非空

扣款費用不計入：
  ▸ 案 202604-038（扣款=$137，專櫃名稱空白）→ 有費用但無專櫃名，不計入
  ▸ 案 1097（birdy yakito，扣款=$50,400，待辦驗）→ 未完成，不計入
```

### 2.6 當月小計還原公式

```
當月小計 = 委外費用 + 維修費用 + 扣款費用（已完成+有專櫃名）
         = $54,425 + $3,116 + $18,755
         = $76,296 ✅
```
