1. 系統概述

合約管理系統（Contract Management System）是集團 Portal 的核心模組之一，涵蓋合約全生命週期管理，從草稿建立、審核生效、請款核銷，到續約申請與到期通知。

2.5 合約明細 Drawer — 操作說明

在合約清單點擊任一列，右側滑出「合約明細 Drawer」。Drawer 提供查看模式與編輯模式，並整合多個功能 Tab，可在不離開清單頁的情況下完成大部分合約操作。

每筆代表一份合約，以 contract_id 為自然主鍵。

7. 分公司費用分攤規則、計算與條件

費用分攤功能允許一份合約的費用由集團內多家分公司共同承擔，支援兩種分攤模式：比例分攤（各公司依百分比）與固定金額分攤（各公司固定承擔金額），並允許兩者混用。

資料儲存方式：費用分攤資料儲存於獨立子表 contract_cost_allocations，採整批覆寫策略（每次儲存時刪除舊記錄再整批插入），公司名稱以字串冗餘儲存（不用 FK），確保歷史分攤記錄不受公司改名或停用影響。

7.1 分攤類型定義

類型代碼中文名稱value 欄位意義範圍使用場景
percentage 比例分攤 各公司承擔的百分比（0–100） 0 ≤ value ≤ 100 各公司按固定比例承擔費用，常見於集團統一採購的服務費分攤
fixed 固定金額分攤 各公司承擔的固定金額（TWD） value ≥ 0 各公司依協議承擔特定金額，常見於廠房租賃、設備維護等固定分攤

7.2 計算公式與範例

▍情境 A：純比例分攤

計算公式：各公司實際負擔金額 = 合約總額（含稅） × 各公司比例 ÷ 100
分攤公司類型比例 value實際負擔（合約總額 $1,200,000）
大直percentage60%$1,200,000 × 60% = $720,000
樂群percentage30%$1,200,000 × 30% = $360,000
總公司percentage10%$1,200,000 × 10% = $120,000
合計100%$1,200,000 ✅
✅ 驗證通過條件：所有 percentage 行的 value 加總 = 100（容差 ±0.01%）

▍情境 B：純固定金額分攤（固定計價合約）

計算公式：各公司固定承擔金額直接記錄，加總必須等於合約總額
分攤公司類型固定金額說明
大直fixed$800,000主要使用方，承擔較高比例
台中fixed$200,000部分共用服務
自由fixed$200,000部分共用服務
合計$1,200,000✅ 等於合約總額
✅ 驗證通過條件（固定計價方式）：所有 fixed 行的 value 加總 = 合約總額（容差 ±1 元，處理四捨五入誤差）
ℹ️ 變動計價（按次收費、依量計費、其他）的合約：固定金額加總不強制驗證，因為合約總額為預估值而非確定值。

▍情境 C：固定金額分攤（變動計價合約）

計價方式 = 按次收費 → 金額變動，固定分攤僅作參考，不驗證加總
分攤公司類型每次固定負擔說明
大直fixed$2,000每次服務大直負擔 $2,000

⚠ 前端狀態面板顯示「計價方式「按次收費」為變動計費，固定金額不強制驗證加總」

7.3 比例與固定金額混用規則

資料表：contract_sla_metrics（指標定義）

欄位類型說明
metric_nameVARCHAR(100)指標名稱（如：系統可用率）
metric_typeVARCHAR(50)類型（可用率/回應時間/解決時間/自訂）
target_valueNumeric(10,4)目標值
target_unitVARCHAR(20)單位（%/小時/天/次/ms）
measurement_periodVARCHAR(20)衡量週期（monthly/quarterly/annual）
is_enabledBoolean是否啟用

資料表：contract_sla_records（達成記錄）

欄位說明
metric_idFK → contract_sla_metrics.id（CASCADE 刪除）
period_label衡量期間標籤（如 2026-01 或 2026-Q1）
actual_value實際達成值
target_value當期目標值（快照，歷史保全）
achieved是否達標（actual_value ≥ target_value 自動判定）
recorded_by登錄人帳號

API 端點

Method路徑說明
GET/contract/{id}/sla-metrics指標清單
POST/contract/{id}/sla-metrics新增指標
PUT/contract/{id}/sla-metrics/{mid}修改指標
DELETE/contract/{id}/sla-metrics/{mid}刪除指標（含所有記錄）
GET/contract/{id}/sla-records?metric_id=達成記錄清單
POST/contract/{id}/sla-records登錄達成記錄（自動判定 achieved）
DELETE/contract/{id}/sla-records/{rid}刪除記錄
GET/contract/{id}/sla-summary各指標達成率統計 + 整體達成率

前端互動設計

  • 整體達成率 + 指標數量 + 未達標指標數 — 頂部三欄 KPI 卡
  • 每個指標顯示環形達成率（≥90% 綠色 / 70~89% 橘色 / <70% 紅色）
  • 折線圖：藍色線 = 實際值；紅色虛線 = 目標基準線；點位顏色：達標綠、未達標紅
  • 尚無指標時顯示空白頁 + 「新增第一個 SLA 指標」引導按鈕

  • 合約搜尋：即時搜尋（輸入關鍵字後呼叫清單 API），每次最多回傳 20 筆候選
  • 「對調」按鈕：交換 A / B 兩份合約位置，不重新搜尋

  • 差異行整列黃底，相同行白底
