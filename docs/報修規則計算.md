## 二、狀態判定邏輯

### 2.1 完成狀態（COMPLETED_STATUSES）

```
已辦驗、已驗收、已結案、結案、完修、已完成、完成
```

> **規則**：`is_completed(status)` 為 **唯一完成判定依據**。`completed_at` 欄位保留供結案天數計算與查閱，**不再用作完成判定條件**。

### 2.2 排除狀態（EXCLUDED_STATUSES）

```
取消
```

> **規則**：排除案件**不計入任何統計 KPI**，但明細總表（報修清單總表）仍完整保留。

### 2.3 三類互斥分類

| 分類 | 條件 | 說明 |
|---|---|---|
| 已完成 | `is_completed(status)` | status ∈ COMPLETED_STATUSES |
| 待辦驗 | `status == "待辦驗"` | 尚未驗收，不歸入已完成也不歸入未完成 |
| 未完成 | 其他（非完成、非待辦驗、非排除） | = 總數 − 已完成 − 待辦驗 |

**三類不重疊，必須滿足：已完成 + 待辦驗 + 未完成 = 本月相關案件總數**

## 三、時間計算規則

### 3.1 統計月份（stat_month）— 決定案件歸屬的核心規則

```
if is_completed(status) AND completed_at is not None:
    stat_year  = completed_at.year
    stat_month = completed_at.month   ← 以「結案月份」為統計月份
else:
    stat_year  = occurred_at.year
    stat_month = occurred_at.month    ← 以「報修月份」為統計月份
```

> **重要**：`stat_month` 決定案件在 Dashboard 費用統計、金額統計 Tab 中落入哪個月。4.1 報修統計 Tab 例外，永遠用 `occurred_at.month`（`occ_month`）。

### 3.2 結案天數（close_days）

```
條件：is_completed(status) AND occurred_at is not None AND completed_at is not None
公式：close_days = (completed_at − occurred_at).total_seconds() ÷ 86400
```

| 結案速度分級 | 條件 |
|---|---|
| 速結 | `close_days ≤ 7 天` |
| 一般 | `7 < close_days ≤ 30 天` |
| 長尾 | `close_days > 30 天` |

### 3.3 樂群 vs 大直的差異

| 模組 | `c.year` / `c.month` 初始值 | `_stat_year` / `_stat_month` 實作 |
|---|---|---|
| **樂群** | `__init__` 已依 status 正確計算 | 直接讀 `c.year` / `c.month` |
| **大直** | `c.year` / `c.month` 永遠 = `occurred_at.year/month` | 先判斷 `is_completed(status)`，再讀 `completed_at.year/month` |

## 四、件數計算公式

### 4.1 Dashboard — 本月相關案件口徑

```
本月相關案件（this_month_cases）
    = 上期累計未結（occurred_at ≤ 前月底 且 NOT is_completed(status)）
    + 本期新增報修（occurred_at.year == year AND occurred_at.month == month）

※ 排除「取消」等 EXCLUDED_STATUSES 案件
```

> 此口徑與 **4.1 報修統計 Tab** 完全對齊。全年檢視（month=0）改用 `filter_cases(all_cases, year, None)`。

### 4.2 Dashboard KPI 計算

| KPI 名稱 | 英文 key | 計算公式 | 資料來源 |
|---|---|---|---|
| 本月相關案件 | `total` | `len(this_month_cases)` | this_month_cases |
| 已完成件數 | `completed` | `count where is_completed(status)` | this_month_cases |
| 待辦驗件數 | `pending_verify` | `count where status == "待辦驗"` | this_month_cases |
| 未完成件數 | `uncompleted` | `total − completed − pending_verify` | 計算值 |
| 平均結案天數 | `avg_close_days` | `mean(close_days) where is_completed AND close_days is not None` | this_month_cases |
| 客房報修件數 | `room_cases` | `count where is_room_case == True` | this_month_cases |
| 花費工時合計 | `total_work_hours` | `sum(work_hours)` | this_month_cases |

### 4.3 4.1 報修統計 Tab — 逐月口徑（永遠用 occ_month）

| 統計項目 | 計算公式 |
|---|---|
| 本月報修件數 | `count where occ_year==y AND occ_month==m` |
| 已完成（完工在 m 月） | `is_completed(status) AND completed_at.year==y AND completed_at.month==m` |
| 已完成（完工截至 m 月底） | `is_completed(status) AND completed_at ≤ (y, m) 月底` |
| 積壓未結（至 m 月底） | `NOT is_completed(status) AND occurred_at ≤ (y, m) 月底` |

## 五、金額計算公式

### 5.1 當月金額（Dashboard 費用摘要）

口徑：`fee_month_cases = filter_cases(all_cases, year, month)`（即 `stat_month == 選定月`）

| 項目 | 公式 | 說明 |
|---|---|---|
| 委外費用（當月） | `sum(outsource_fee)` | fee_month_cases |
| 維修費用（當月） | `sum(maintenance_fee)` | fee_month_cases |
| 扣款費用（當月） | `sum(deduction_fee)` | fee_month_cases |
| 當月小計 | 委外 + 維修 + 扣款費用 | |
| 扣款專櫃家數（當月） | 唯一店名數（見第六章） | fee_month_cases，僅計已完成案件 |

### 5.2 YTD 累計費用（Dashboard 費用 KPI 卡）

口徑：`ytd_cases`（月份選定時：`stat_year==year AND stat_month ≤ 選定月`；全年：全年度）

| 項目 | 英文 key | 公式 |
|---|---|---|
| 委外+維修費用（YTD） | `annual_fee` | `sum(outsource_fee) + sum(maintenance_fee)` |
| 扣款費用（YTD） | `annual_deduction_fee` | `sum(deduction_fee)` |
| 扣款專櫃家數（YTD） | `annual_deduction_counter` | 唯一店名數，跨月去重（僅計已完成） |
| 扣款費用合計（YTD） | `annual_counter_fee` | `sum(deduction_fee)` for 已完成且有專櫃名稱 |

### 5.3 金額統計 Tab — 12 個月交叉表

對每個月 `m ∈ [1, 12]`，口徑同 `fee_month_cases`（stat_month == m）：

| 費用項目 | 計算方式 |
|---|---|
| 委外費用 | `sum(outsource_fee)` |
| 維修費用 | `sum(maintenance_fee)` |
| 扣款費用 | `sum(deduction_fee)` |
| 扣款專櫃家數 | 唯一店名數，**僅計已完成案件** |
| 月份小計 | 委外 + 維修 + 扣款費用（家數欄不加入小計） |
| 全年唯一家數 | 跨月去重後唯一店名數 |

## 六、扣款專櫃計算規則

### 6.1 計入條件（三個必要條件）

```
1. is_completed(status)           ← 已完成案件
2. deduction_fee > 0              ← 有扣款金額
3. deduction_counter_name != ''   ← 有填寫專櫃名稱
```

> **重要**：加上 `is_completed` gate 的原因：待辦驗案件的 `c.month = occurred_at.month`，若不加此限制，這些案件的 `completed_at`（可能為未來月份）會污染當月/YTD 統計。

### 6.2 多櫃（"多櫃"）解析邏輯

```
if deduction_counter_name == "多櫃":
    從 finance_note 按行解析（格式：店名 金額）
    去除結尾數字後取店名 → counter_stores list
else:
    counter_stores = [deduction_counter_name]
```

### 6.3 唯一家數計算

```
# 當月（Dashboard 當月金額 / 金額統計 Tab 逐月）
unique_stores = set()
for case in fee_month_cases where 三個必要條件:
    unique_stores.update(case.counter_stores or [case.deduction_counter_name])
monthly_count = len(unique_stores)

# YTD（Dashboard KPI 卡）
unique_stores_ytd = set()
for case in ytd_cases where 三個必要條件:
    unique_stores_ytd.update(case.counter_stores or [case.deduction_counter_name])
ytd_count = len(unique_stores_ytd)   ← 跨月去重，不得直接加總各月家數
```

## 七、各口徑對比總表

| 統計維度 | 月份分桶依據 | 案件範圍 | 完成判定 |
|---|---|---|---|
| Dashboard 本月相關案件 | `occ_month`（報修月） | 上期未結 + 本期新增 | `is_completed(status)` |
| Dashboard 費用（當月） | `stat_month`（完工月/報修月） | `stat_month == 選定月` | — |
| Dashboard 費用（YTD） | `stat_month` | `stat_month ≤ 選定月` | — |
| Dashboard 扣款專櫃（YTD） | `stat_month` | `stat_month ≤ 選定月` | **is_completed 必要** |
| 金額統計 Tab（逐月） | `stat_month` | `stat_month == m` | 扣款專櫃欄：is_completed 必要 |
| 4.1 報修統計 Tab | `occ_month`（報修月） | `occ_month == m` | `is_completed(status)` |
| 4.2 結案時間 Tab | `completed_at.month` | `is_completed AND completed_at in (y,m)` | `is_completed(status)` |

## 八、資料流程圖

```
Ragic API
    │
    ▼
fetch_all_cases()           ← 取全部案件（含 stale-while-revalidate cache）
    │
    ▼
RepairCase.__init__()       ← 清洗、標準化、計算衍生欄位
    │  - stat_year/stat_month（依 status 決定）
    │  - is_completed_flag, is_excluded_flag
    │  - close_days
    │  - counter_stores（多櫃解析）
    │
    ▼
compute_dashboard(all_cases, year, month)
    │  ├─ this_month_cases → KPI 件數
    │  ├─ fee_month_cases  → 當月費用、扣款專櫃（當月）
    │  └─ ytd_cases        → 累計費用、扣款專櫃（YTD）
    │
    ├─ compute_repair_stats()    → 4.1 報修統計
    ├─ compute_closing_time()    → 4.2 結案時間
    ├─ compute_type_stats()      → 4.3 報修類型
    └─ compute_fee_stats()       → 金額統計 Tab
```

## 九、特殊情況與邊界條件

| 情況 | 處理方式 |
|---|---|
| `completed_at` 為空但 status 已完成 | `stat_year/stat_month = None`，不計入任何月份 bucket |
| `occurred_at` 為空 | 排除 close_days 計算；`occ_year/occ_month = None` |
| status 為排除狀態（取消） | `is_excluded_flag = True`，所有統計函式前先過濾 |
| 大直模組 `deduction_counter` | 大直無此欄位，固定為 0；所有扣款專櫃邏輯僅適用於樂群 |
| 花費工時為 0 | 嘗試讀取「工務處理天數」換算（×24 hr；上限 365×24 防誤植） |
| 多櫃且財務備註為空 | `counter_stores = ["多櫃"]`（當作一家店） |
